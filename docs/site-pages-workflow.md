# Работа со страницами в Directus Studio

Цель этого workflow: редактор начинает не с технической таблицы секций, а с
понятной страницы сайта.

## Основной сценарий

1. Откройте `Страницы сайта`.
2. Выберите страницу: `home`, `store`, `trade`, `passport` или `club`.
3. Внутри карточки страницы откройте блок `Секции страницы`.
4. Правьте текст, кнопки и главное изображение секции.
5. Для временного скрытия блока выключите `Показывать на сайте`.

`Slug`, `template`, `Ключ блока`, `Тип блока` и `JSON-настройки блока` связаны с
Next renderer. Их лучше не менять без проверки разработчиком.

## Быстрые закладки

Setup добавляет редакторские bookmarks:

- `Опубликованные страницы`
- `Черновики страниц`
- `Главная`
- `Store`
- `Trade`
- `Passport`
- `Club`
- `Скрытые секции`

Закладки по страницам находятся в коллекции `Секции страниц` и показывают только
нужные блоки в правильном порядке.

## Изображения

Для секций используйте поле `Главное изображение блока`. Файлы должны лежать в
папке `ISVOI Site Assets`.

Товарные фото не нужно добавлять в секции страниц. Они живут отдельно в
`Устройства` → `Фото устройства`.

## Применение setup

```bash
npm run directus:setup:site-pages-workflow
```

На production скрипт применяется тем же способом, что и остальные
`directus:setup:*`: его SQL нужно передать в PostgreSQL контейнер Directus.
